# AP Cybersecurity Course Framework

This document provides a detailed breakdown of the AP Cybersecurity course framework based on the official Course and Exam Description (CED, effective Fall 2026).

## Course Skills

The course is organized around four skill categories. Each skill is developed across all five units.

| Skill | Category | Description |
|---|---|---|
| **1.A** | Analyze Risk | Identify vulnerabilities, threats, and attack methods, and explain how they generate risk |
| **1.B** | Analyze Risk | Determine ways adversaries exploit vulnerabilities to compromise an asset |
| **1.C** | Analyze Risk | Evaluate the likelihood and impact of risks |
| **1.D** | Analyze Risk | Document the likelihood and impact of risks |
| **2.A** | Mitigate Risk | Identify security controls and explain how they mitigate risks |
| **2.B** | Mitigate Risk | Determine layered security controls to address vulnerabilities |
| **2.C** | Mitigate Risk | Evaluate the impact of protective risk-management strategies |
| **2.D** | Mitigate Risk | Implement and log mitigations |
| **3.A** | Detect Attacks | Identify methods for monitoring systems and explain how they detect attacks |
| **3.B** | Detect Attacks | Determine strategies and methods to detect attacks |
| **3.C** | Detect Attacks | Evaluate the impact of threat detection methods |
| **3.D** | Detect Attacks | Detect and classify cyberattacks by analyzing digital evidence |
| **4.A** | Collaborate | Develop clear, shared team objectives related to a cybersecurity task |
| **4.B** | Collaborate | Determine clear roles and responsibilities for team members |
| **4.C** | Collaborate | Implement AI as a collaboration tool individually and as a group |
| **4.D** | Collaborate | Complete assigned work to accomplish a collaborative cybersecurity task |

> **Note on AI:** Several skills explicitly mention "with and without the support of AI," reflecting the course's integration of AI as both a threat and a defensive tool.

## Unit Details

### Unit 1: Introduction to Security (~10 class periods)

**Focus:** Personal security in everyday life. Students learn foundational concepts: threats, vulnerabilities, risk, adversaries, social engineering, and the emerging role of AI in both attacks and defense.

| Topic | Title | Key Learning Objectives |
|---|---|---|
| 1.1 | Understanding Social Engineering | Identify indicators of social engineering; explain how tactics generate risk; describe impacts on victims |
| 1.2 | Suspicious Website Logins | Identify signs of password attacks; explain how adversaries exploit weak authentication; explain how to strengthen authentication |
| 1.3 | Best Practices for Public Networks | Identify adversary types and wireless attack types; explain how individuals can protect themselves on public Wi-Fi |
| 1.4 | AI-Based Cybersecurity Attacks | Explain how adversaries use AI-powered tactics; explain how to protect against AI-assisted attacks |
| 1.5 | Leveraging AI in Cyber Defense | Explain how defenders can leverage AI; explain how AI tools enable new defensive capabilities |

---

### Unit 2: Securing Spaces (~21 class periods)

**Focus:** Physical security in organizational contexts. Students move from personal security to protecting physical spaces — understanding physical vulnerabilities, protective measures, and detection methods.

| Topic | Title | Key Learning Objectives |
|---|---|---|
| 2.1 | Cyber Foundations | Identify social engineering attacks and adversary types; describe cyberattack phases, risk assessment process, and risk management strategies |
| 2.2 | Physical Vulnerabilities and Attacks | Identify physical vulnerabilities and attack methods; explain how adversaries exploit them |
| 2.3 | Protecting Physical Spaces | Identify physical security controls; explain how they mitigate risks |
| 2.4 | Detecting Physical Attacks | Identify methods for monitoring physical spaces; explain how they detect attacks |

---

### Unit 3: Securing Networks (~26 class periods)

**Focus:** Network vulnerabilities, protective controls (managerial, segmentation, firewalls), and network attack detection.

| Topic | Title | Key Learning Objectives |
|---|---|---|
| 3.1 | Network Vulnerabilities and Attacks | Identify network vulnerabilities and attack methods; explain how they generate risk |
| 3.2 | Protecting Networks: Managerial Controls and Wireless Security | Identify managerial controls and wireless security measures; explain how they mitigate risks |
| 3.3 | Protecting Networks: Segmentation | Identify network segmentation controls; explain how they mitigate risks |
| 3.4 | Protecting Networks: Firewalls | Identify firewall controls; explain how they mitigate risks |
| 3.5 | Detecting Network Attacks | Identify network monitoring methods; explain how they detect attacks; analyze network logs |

---

### Unit 4: Securing Devices (~23 class periods)

**Focus:** Device vulnerabilities, authentication, device protection, and attack detection on devices.

| Topic | Title | Key Learning Objectives |
|---|---|---|
| 4.1 | Device Vulnerabilities and Attacks | Identify device vulnerabilities and attack methods; explain how they generate risk |
| 4.2 | Authentication | Identify authentication methods; explain how they mitigate risks |
| 4.3 | Protecting Devices | Identify device protection controls; explain how they mitigate risks |
| 4.4 | Detecting Attacks on Devices | Identify device monitoring methods; explain how they detect attacks; analyze device evidence |

> 💡 **FRQ Connection:** The free-response question (Device Security Analysis) is directly tied to this unit's content. Students analyze security policies, firewall configurations, file-system permissions, and log files for a single device.

---

### Unit 5: Securing Applications and Data (~30 class periods)

**Focus:** The largest unit. Application and data vulnerabilities, access controls, cryptography (symmetric and asymmetric), application protection, and attack detection.

| Topic | Title | Key Learning Objectives |
|---|---|---|
| 5.1 | Application and Data Vulnerabilities and Attacks | Identify application/data vulnerabilities and attack methods; explain how they generate risk |
| 5.2 | Managerial Controls and Access Controls | Identify managerial and access controls; explain how they mitigate risks |
| 5.3 | Protecting Stored Data with Cryptography | Identify cryptographic methods for data at rest; explain how they mitigate risks |
| 5.4 | Asymmetric Cryptography | Identify asymmetric encryption methods; explain how they work and mitigate risks |
| 5.5 | Protecting Applications | Identify application protection controls; explain how they mitigate risks |
| 5.6 | Detecting Attacks on Data and Applications | Identify monitoring methods; explain how they detect attacks; analyze evidence |

## Legal & Regulatory Context

The course integrates laws, industry regulations, and professional norms throughout, including:

- **PII** (Personally Identifiable Information)
- **PHI** (Protected Health Information)
- **Student education records** (FERPA)
- **Financial records and transactions**
- **PCI DSS** (Payment Card Industry Data Security Standard)

Instructors are expected to introduce these concepts and integrate them into appropriate topics.

## Exam Structure

| Section | Type | Questions | Weight | Time |
|---|---|---|---|---|
| I | Multiple-Choice | 60 | 70% | 80 minutes |
| II | Free-Response (Device Security Analysis) | 1 | 30% | 50 minutes |

**MCQ Skill Weighting:**
- Skill Category 1: Analyze Risk — 25–40%
- Skill Category 2: Mitigate Risk — 25–40%
- Skill Category 3: Detect Attacks — 25–40%

**FRQ (Device Security Analysis):**
Students receive simulated sources about a single digital device:
- Security policies
- Firewall configurations
- File-system permissions
- Log files

They must: identify security issues, detect evidence of attacks, describe how configuration/permission changes affect the device and users, and evaluate how security controls influence network traffic and device behavior. Students must cite evidence and explain reasoning.

**FRQ Task Verbs:**

| Verb | Meaning |
|---|---|
| Identify | Provide information about cybersecurity concepts or evidence from given sources |
| Explain | Provide reasons that support a solution, using specific evidence to validate conclusions |
| Describe | Provide information about a cybersecurity process or outcome |
| Determine | Provide a specific result by applying appropriate criteria or reasoning |
| Write | Express a proper command that has the indicated effect |
